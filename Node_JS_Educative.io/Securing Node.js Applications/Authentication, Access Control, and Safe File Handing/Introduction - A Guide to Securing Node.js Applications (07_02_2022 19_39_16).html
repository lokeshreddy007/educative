Storytime#

Erica works for a small manufacturing company as a programmer and general IT person. She was recently tasked with developing software so the office staff can move from paper records to digital.

The first step was to ask staff members to scan documents when they were finished with them. Instead of filing the paper forms, they are now stored electronically.

The development of the archiving system proceeded well. The system consisted of a basic database that lists all documents with various filters like department, date, and tags. Nothing too complicated; Erica wanted to keep it simple.

The documents displayed to users are filtered based on their department and position. Everything worked well for several months. But then there was a breach of a secure document. The subsequent investigation determined that a disgruntled employee gained access to a file meant only for senior management. The employee leaked the document to a competitor to secure a comfortable position for themselves at that company.

So how did it happen? Erica has been working all weekend trying to figure it out, when boom, she finds it. Its so obvious, but she never thought to secure it.

In Ericas system, files are uploaded and named after the ID fields of their respective database records. An uploaded PDF might be named 13424.pdf, while the next file, a DOC, would be named 13425.doc, and so on. Erica secured the document listing that each user sees, but had not secured the files themselves when opened or downloaded by the browser. There was no way to stop a user from gaining access to sensitive documents by simply guessing filenames by incrementing the ID.

The stories are getting a little long, but if youre committed enough, you can make any story work. I once told a woman I was Kevin Costner, and it worked because I believed it.
